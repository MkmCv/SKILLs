## AI 主持人应重点关注哪些方面？

**应用场景**：新功能介绍视频，面向产品/商务/非技术人员

**讲解节奏与时间控制**：紧凑节奏
- 开头快速切入（前5秒明确变化）
- 短句、快速过渡、不重复
- 每句话都必须带新信息
- 减少主持人互动铺垫，直入主题

**专注于的来源**：
- 重点讲解 Sources 中的"新增功能/变化点"与"价值"
- 如果有对比图/流程图：优先展示，确保画面清晰可读

**节目结构**：
1. 开场：一句话说明新增了什么能力
2. 问题与价值：以前哪里不方便？现在为什么更好？
3. 使用方式：用"你可以…/现在只需要…"描述
4. 结尾：一句话总结价值（效率/成本/风险/体验）

**画面要求**（如果 Sources 中有图片/截图）：
- **图片最大化**：全屏显示，或至少占画面 80%
- **关键信息高亮**：UI 元素/按钮文案/对比点用色块/箭头/高亮标注
- **页面布局合理**：信息层级清晰（标题/正文/重点用字号/颜色区分）、避免信息密集堆砌（一屏最多 3-5 个信息点）、留白与呼吸感、文字排版左对齐或居中
- **可读性硬要求**：如果关键 UI 元素/文案在画面中不可读，则该步骤视为失败，必须放大图片或裁剪到关键区域或拆分成两个镜头
- **旁白约束**：旁白不要补充图片之外的 UI 文案/按钮名称/数值

**参考时间轴框架**（以紧凑节奏 3 分钟为例）：
- 0-15s：全屏展示"新增了什么能力"，快速切入
- 15-60s：问题场景（以前哪里不方便？为什么需要改进？）
- 60-150s：新方式演示（用"你可以…"描述，配对比图/流程图）
- 150-180s：价值总结（效率/成本/风险/体验哪个被改善）

**禁止内容**：技术原理、代码、命令、参数细节
